The OpenMC data downloader offers a command line tool and a python API. Both interfaces allow the downloading of preprocessed h5 cross section files. There is a pie pie installer for the package and users can install the software with the command, pip install openmc data downloader. Once installed the command line tool can be used. All the options can be found using the minus minus help argument. A library or libraries are always needed and these should be provided in order of preference. As you can see there are several options available including jeff 3.2, fendl 3.1d, tendl 2019, WMP and endf. The downloader will try to find isotopes from the libraries provided. If multiple libraries are provided then they will be searched preferentially in the input order. Isotopes are downloaded and assigned filenames based on their library and isotope. Additionally a cross sections xml file will also be made which allows use of the h5 files in OpenMC This allows users easily to create a fully reproducible nuclear data environment. You can download specific isotopes from specific libraries using minus l for library and minus i for isotope. Running the same command again will detect that the cross sections have been downloaded and skip the downloading. Elements can also be requested using the minus e for elements argument, and will automatically be expanded to their natural abundances. Elements and isotopes can both be specified by combining minus e and minus i arguments. The h5 files can be sent to a specific directory using the minus d argument. By default the h5 files for neutrons are downloaded, but different particles, or a combination of particles can be specified using the minus p for particles argument. Acceptable options are photon and neutron. The package can also be imported into a python script and used in conjunction with OpenMC to download the nuclear data required for the simulation. In this example we see that OpenMC data download has been imported as odd. An OpenMC material has been made and that material has been passed to the just in time library generator with a preference for the fendl nuclear data library. Once run, this python script will now download the nuclear data required for simulating with this material. The just in time download can also be called with a list of material objects. Also an OpenMC materials object can be used. Additionally Material objects and Multi Material objects from the neutronics material maker can also be used. The OpenMC cross section enviromental varible is automatically set to the path of the new cross section.xml that has been generated on the fly. This means that launching a simulation from the same python script tells OpenMC to make use of the recently downloaded cross sections.
